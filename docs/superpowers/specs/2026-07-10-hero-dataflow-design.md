# Hero Section 垂直数据流瀑布设计文档

## 项目背景
当前 hero section 中间区域视觉上比较空旷，需要增加赛博朋克风格的被动动画装饰元素来填充空间，提升科技感和视觉丰富度。

## 设计目标
- 填充页面中间空白区域，使整体视觉更饱满
- 保持与现有赛博朋克/科技风格一致
- 纯被动动画，不需要鼠标交互
- 半透明效果，不遮挡文字和主要内容
- 性能友好，不影响页面流畅度

## 整体布局
- **位置**：页面中央偏上区域，文字内容右侧、人物图片左侧
- **面积**：约占页面宽度的 30-40%，高度 60-70vh
- **层级**：z-index 介于文字（最上层）和人物图片之间
- **透明度**：整体较低，不抢主视觉

## 核心组件

### 1. 垂直数据流（Vertical Data Streams）
- **数量**：6-8 条，分布在左右两侧
- **内容**：01二进制数字、十六进制字符、科技词汇（SYSTEM, DATA, SIGNAL, QUANTUM, NEURAL, PULSE等）
- **动画**：从上往下滚动，不同流有不同速度（快/中/慢）
- **透明度**：从 10% 到 40% 不等，形成层次感
- **颜色**：主色调橙黄色 (#FFAA66, #FF8A33)，偶尔有白色高亮字符
- **边缘处理**：顶部和底部渐变透明，无硬边

### 2. 中央全息数据面板（Holographic Data Panel）
- **位置**：数据流区域中央
- **形状**：矩形面板，带发光边框和四角装饰
- **内容**：
  - 顶部：系统状态标题（SYS.STATUS: ONLINE）
  - 中间：3-4 行伪数据读数滚动
  - 底部：进度条、简易波形图、柱状图（CSS动画）
- **特效**：
  - 扫描线效果
  - 偶尔的故障闪烁（glitch）
  - 边框发光脉动
  - 半透明背景

### 3. 装饰元素
- 横向扫描线穿过数据流区域（慢速上下移动）
- 偶尔有数据粒子从数据流中迸出（少量）
- 底部淡淡的光晕投影效果

## 与现有元素的关系
- **现有矩阵雨 (matrix rain)**：保留，作为背景层（更淡、更大、z-index更低），新增的数据流在更上层、更精细
- **人物图片**：数据流在人物左侧，不会覆盖人物
- **文字内容**：文字始终在最上层（z-index: 3+），清晰可读
- **现有装饰效果**：保留所有现有效果（粒子、雷达、齿轮、闪电等），数据流融入其中

## 技术实现
- 纯 CSS + 原生 JavaScript 实现
- 数据流使用 DOM 元素 + CSS animation（性能更好）
- 数据面板使用 CSS 动画
- 响应式：移动端隐藏或简化

## 非目标
- 不添加鼠标交互（纯被动动画）
- 不添加游戏化元素
- 不影响页面性能
- 不改变现有布局结构
