# 中文代码注释助手

## 描述
这是一个代码注释规范技能，确保所有生成的代码都带有清晰、详细的中文注释。无论是新生成的代码还是修改的代码，都必须遵循本注释规范。

## 使用场景
- 用户要求生成或修改任何代码时
- 代码需要被团队其他人阅读和维护时
- 初学者学习代码时

## 指令

### 1. 函数/方法注释
每个函数或方法上方必须有中文注释块，包含：
- **功能说明**：一句话说清楚这个函数做什么
- **参数说明**：每个参数的含义和类型
- **返回值说明**：返回什么内容和类型
- **示例**（可选）：简单的使用示例

格式参考：
```python
def calculate_average(numbers):
    """
    计算数字列表的平均值
    
    参数:
        numbers (list): 包含数字的列表，如 [1, 2, 3, 4, 5]
    
    返回值:
        float: 列表中所有数字的平均值
    
    示例:
        >>> calculate_average([1, 2, 3])
        2.0
    """
    return sum(numbers) / len(numbers)
```

### 2. 行内注释
对于复杂的代码行或关键逻辑，必须在行后添加中文注释解释：
- 复杂的计算逻辑
- 不容易一眼看懂的语法
- 业务逻辑的关键节点
- 特殊处理的原因

### 3. 变量注释
重要的变量声明时，添加注释说明变量的用途：
- 存储什么数据
- 数据格式是什么
- 为什么需要这个变量

### 4. 注释风格要求
- 注释语言：**必须是中文**
- 注释语气：简洁明了，不说废话
- 注释位置：
  - 函数注释：函数定义行的上方
  - 行内注释：代码行的右侧，空两格后写
  - 变量注释：变量声明行的上方或右侧

### 5. 不同语言的注释格式
- **Python**：使用三引号 `"""..."""` 作为文档字符串
- **JavaScript/TypeScript**：使用 `/** ... */` 作为 JSDoc 注释
- **Java**：使用 `/** ... */` 作为 Javadoc 注释
- **C/C++/C#**：使用 `///` 或 `/** ... */` 注释
- **HTML/CSS**：使用 `<!-- 注释内容 -->`

## 注意事项
- 不要为了注释而注释，简单明显的代码可以少注释
- 注释要准确，不能和代码逻辑矛盾
- 代码修改后，记得同步更新注释
- 优先保证注释的质量，而不是数量
